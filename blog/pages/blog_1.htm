My first programming language

Pascal was the first programming language that I had access to. It's really great because of its simplicity and ease of access to those new to programming. So today I'm going to analyze what I like and dislike about the first programming language of many of these guys.

Things I like


Each variable has an explicit data type

Unlike Python or JavaScript, every time we declare a variable, we must declare its data type. This makes the variable always unified in the data type and does not encounter the situation of easily changing the data type, which leads to accuracy when executing programs.


The syntax is simple and easy to read

Pascal is a very accessible language. For example:


Instead of having to remember some kind of placeholder and add & before each variable when using scanf to input data from the keyboard, we just call read or readln and then pass the variables that we need to enter is complete.Instead of using {} to add a script we want, which doesn't make much sense in the language we use, Pascal uses the begin and end keywords to add the script we want, which This will make our code more readable.


Working with strings of characters is not complicated

If you have ever studied the C programming language, you will see that manipulating strings is very complicated. However, Pascal is much simpler than C. Instead of using strcpy as in C to assign a new string to the string, we only need to perform the normal assignment in Pascal or instead of using the strcat function in C to perform the addition of two strings of characters together, we addition can be used to concatenate not only two strings, but also multiple strings.


Things I don't like


The variable must be declared after the var keyword

This is what I like most in Pascal and many other old languages. The declaration at the beginning of the program brings a great inconvenience. Every time you are programming a program and forget to declare a variable, it needs to go back to the beginning of the code to find the var keyword to declare it. With short and simple code, it's okay but when your code becomes complicated, it will cause a lot of trouble for you.


It is a case-insensitive programming language

This is not a big problem, but such a distinction will cause programmers (especially beginners) to write code that sometimes lacks consistency and makes the code become confused.


It is an old and uncommon programming language at the moment

The lack of popularity will make programming work much more difficult because you lack a lot of community support and it is difficult to use Pascal to get a job later.


In short, although Pascal learning may not help you get a job later or write a program for yourself, but Pascal is a programming language that is worth to experience especially beginners. early learn to program. Hopefully the article will help you get a little suggestion to be able to start writing the first code for yourself.
